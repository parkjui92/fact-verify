# 한국 학술·정책 검증원 접근 가이드

fact-verify의 Step 3-D(한국 학술·정책 출처 검증)가 사용하는 검증원과, 정밀 검증을 위한 공공데이터포털 인증키 설정 방법.

## 접근 등급

| 등급 | 조건 | 할 수 있는 것 |
|------|------|---------------|
| **무키(기본)** | 아무 설정 없음 | 각 포털 공개 검색 페이지 WebFetch + CrossRef(무키). 소량 확인 가능 |
| **유키(권장)** | 공공데이터포털 인증키 설정 | KCI·RISS·국회도서관 REST API로 정밀·대량 조회. 서지 필드 구조화 대조 |

키는 **요구사항이 아니라 성능 옵션**이다. 없으면 없는 대로 공개 검색 범위에서 확인하고, 못 미친 부분은 `NEEDS_REVIEW`로 남긴다. 없는 능력을 있는 척하지 않는다.

- **PRISM** 정책연구관리시스템 — prism.go.kr (정부 정책연구용역)

## 인증키 발급 (공공데이터포털 경로 — 권장)

1. [공공데이터포털](https://www.data.go.kr) 회원가입
2. 위 표의 각 데이터셋 페이지에서 **[활용신청]** — 대부분 자동승인(즉시~수 시간)
3. 마이페이지 > 인증키(일반 인증키, Encoding/Decoding) 확인

> KCI 포털(kci.go.kr) 직접 발급 경로도 있으나 **재단 공문 발송**이 필요해 더 느리다. 공공데이터포털 경로를 권장한다.

## 키 설정 (fact-verify가 읽는 곳)

환경변수에 넣는다(예시). 스킬은 이 값이 있으면 API 경로로, 없으면 공개 검색 경로로 자동 분기한다.

```bash
# ~/.zshrc 등
export DATA_GO_KR_KEY="발급받은_Decoding_인증키"
```

> ⚠️ 인증키는 비밀값이다. 저장소·문서·로그에 노출하지 않는다. fact-verify는 키 자체를 출력하지 않으며, 검증 시 외부로 전송하는 것은 조회 파라미터(제목·저자·연도)뿐이다.

## 응답 처리 원칙

- API/검색 결과가 **정확 일치**: 서지 대조 후 `VERIFIED`. 인용과 연도·권호가 다르면 `VERIFIED`(실존) + 오기 지적.
- **제목 부분일치 후보만** 있음: `NEEDS_REVIEW`, 후보 제시.
- **국내외 어디에도 없음**: `REJECT`(환각 출처 의심).
- **키·네트워크 없어 확인 불가**: `NEEDS_REVIEW`(확인 불가), 오류로 단정하지 않음.
